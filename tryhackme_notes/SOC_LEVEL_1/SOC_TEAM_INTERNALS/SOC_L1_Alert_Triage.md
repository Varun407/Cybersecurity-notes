## SOC Roles

### L1 Analyst

* Reviews alerts
* Performs initial triage
* Determines TP/FP
* Escalates when needed

### L2 Analyst

* Deep investigation
* Incident response
* Remediation

### SOC Engineer

* Creates and tunes detection rules
* Improves alert quality

### SOC Manager

* Tracks alert handling performance

# Important Alert Fields

## Alert Time

Time when alert was generated.

> Alert time may be later than actual event time.

---

## Alert Name

Summary of detected activity.

Examples:

* Unusual Login Location
* Email Marked as Phishing
* RDP Bruteforce

---

## Severity

| Severity | Priority |
| -------- | -------- |
| Critical | Highest  |
| High     | High     |
| Medium   | Moderate |
| Low      | Lowest   |

---

## Status

* New
* In Progress
* Closed

---

## Verdict (Classification)

### True Positive (TP)

Real malicious activity.

### False Positive (FP)

Legitimate activity incorrectly flagged.

---

## Assignee

Analyst responsible for the alert.

---

## Description

Usually contains:

* Detection logic
* Why it is suspicious
* Investigation guidance

---

Useful investigation data such as:

# Alert Prioritisation

### 1. Filter Alerts

Ignore alerts that are:

* Already assigned
* Already closed
* Being investigated

Take only new alerts.

---

# Investigation Checklist

### Understand Scope

Identify affected:

* User
* Host
* Server
* Cloud Asset
* Website

### Understand Activity

Examples:

* Suspicious login
* Malware execution
* Phishing
* Data exfiltration

### Review Context

Check:

* Events before alert
* Events after alert
* Related alerts

### Verify Findings

Use:

* Threat intelligence
* Internal documentation
* Security tools

---

# Final Actions

## If Malicious

* Mark as True Positive
* Escalate if required

## If Benign

* Mark as False Positive

## Always

* Add investigation notes
* Explain reasoning
* Close alert

---

# Key Points

* Alerts are generated from logs, not directly from events.
* L1 analysts perform initial triage and classification.
* Severity determines investigation priority.
* Investigate oldest alerts first when severity is equal.
* TP = Real threat.
* FP = Benign activity flagged as malicious.
* Assign alert before investigating.
* Change status to In Progress during investigation.
* Document findings before closing.
