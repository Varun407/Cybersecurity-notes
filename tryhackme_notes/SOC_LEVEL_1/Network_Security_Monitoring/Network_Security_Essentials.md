# Network Security Essentials — Complete Notes

# 1. Network Overview

## What is a Network?

A computer network is an organized structure where network assets connect to enable:
- Communication between devices
- Resource sharing
- Connectivity with each other and the world

## Why Network Security Matters

**The Network Perimeter:**
- Segregates internal ecosystem from external Internet
- Often the first target for attackers
- Critical for early attack detection

**Key Concept:** You can't defend what you can't see.

---

# 2. Network Components

## User Workstations (Endpoints)

**Purpose:** Employees do daily work (PCs, laptops)

**Security Risk:**
- Most common entry point for attackers
- Phishing emails
- Malicious downloads

**Example:** Phishing email drops malware on a finance user's PC

**Why It Matters:**
- Endpoints are often less monitored
- Compromise can give attackers foothold to move laterally
- Endpoint logs reveal malicious processes
- Network logs show C2 (Command & Control) connections

## File & Database Servers

**Purpose:**
- **File Servers:** Centralized access to shared documents
- **Database Servers:** Manage structured data (customer records, HR, financial data)

**Security Risk:**
- Attackers target these for valuable/sensitive data
- Ransomware operators target file servers
- Data exfiltration campaigns target these servers

**What to Monitor:**
- Unusual access patterns
- Data transfer volumes
- Authentication attempts

## Application Servers (Web, Email, VPN)

**Purpose:**
- **Web Servers:** Host company websites and web applications
- **Email Servers:** Handle corporate communications
- **VPN Gateways:** Allow secure remote access

**Security Risk:**
- Externally facing → High-value targets
- Constantly scanned for vulnerabilities
- Weak configurations exploited

**What to Monitor:**
- Application logs
- Firewall alerts
- IDS signatures

**Common Attacks:**
- SQL injection on web apps
- Brute-force login attempts
- Suspicious external IPs

## Active Directory (AD) / Authentication Servers

**Purpose:**
- Identity backbone of enterprise networks
- Manages users, groups, computers, and access rights

**Security Risk:**
- Main target for privilege escalation
- Persistence and lateral movement
- Single compromised domain admin = entire enterprise compromised

**What to Monitor:**
- Multiple failed login attempts (password spraying)
- Unusual logins from external IPs
- Logins at odd hours
- Accounts accessing unusual systems

## Routers & Switches

**Purpose:**
- **Routers:** Connect different networks (LAN to Internet)
- **Switches:** Connect devices within same network

**Security Risk:**
- If compromised, attackers can:
  - Intercept traffic (Man-in-the-Middle)
  - Create backdoors
  - Reroute traffic
  - Open hidden channels to Internet

## Firewalls / Perimeter Devices

**Purpose:**
- Primary security gateway
- Controls traffic between trusted internal and untrusted Internet
- Inspects incoming/outgoing packets
- Decides to allow or block based on rules

**Modern Firewall Capabilities:**
- Deep inspection of applications
- Intrusion prevention
- Malware detection

**Importance:**
- Protects from direct Internet exposure
- Prevents unauthorized access
- Logs every connection attempt

# 3. Network Visibility

## Why Visibility is Crucial

**Analogy:** Securing a house with no windows or security cameras

**Without Visibility:**
- Malware infections go unnoticed
- Unauthorized access undetected
- Data exfiltration invisible

**With Visibility, Analysts Can:**
- Detect anomalies
- Investigate incidents
- Hunt for threats
- Ensure compliance

## Two Main Categories of Log Sources

### Host-Centric Logs

**Generated By:** Individual devices (hosts)

**View:** Ground-level view of specific machine

**Key Sources:**

| Source | Description |
|--------|-------------|
| Operating System Logs | Windows Event Logs, Linux syslog, macOS logs |
| Application Logs | Web servers, databases, applications |
| Security Tool Logs | Antivirus, EDR, HIDS |

**What They Answer:**
- Detailed forensic analysis
- Process and execution tracking
- User activity monitoring
- Malware impact assessment

**Examples:**
- User logons
- Process creation
- Service startups
- Failed login attempts
- File access/modification

### Network-Centric Logs

**Generated By:** Network appliances

**View:** What's happening between devices

**Key Sources:**

| Source | Description |
|--------|-------------|
| Firewalls | Allowed/blocked connections |
| IDS/IPS | Known attack signatures, anomalies |
| Routers/Switches | Flow data (who talked to whom) |
| Web Proxies | Website visits, web-based threats |
| VPN | Remote access connections |

**What They Answer:**
- Early threat detection
- Command & Control (C2) identification
- Lateral movement tracking
- Data exfiltration detection
- Broad attack context

**Examples:**
- Source/destination IPs
- Ports and protocols
- Action taken (allowed/blocked)

**Key Principle:** Correlate host and network logs for complete picture
> Host-centric logs = What happened inside a room
> Network-centric logs = Who entered and left the building

# 4. Network Perimeter

## What is the Network Perimeter?

The boundary separating:
- **Internal Network (Trusted Zone):** Employees, servers, business applications
- **External Internet (Untrusted Zone):** Potential threats

**Analogy:** The main gate or front door of a secure building

## Why the Perimeter Matters

**Attackers always start probing from outside**

The perimeter is:
- First line of defense
- First place SOC analysts see malicious activity

**If Perimeter is Weak/Misconfigured:**
- Exploit exposed services (RDP, MySQL, SMB)
- Perform scanning and reconnaissance
- Launch brute-force attacks
- Exfiltrate data

## Common Perimeter Components

| Component | Purpose |
|-----------|---------|
| **Firewalls** | Gatekeepers filtering traffic |
| **Routers/Gateways** | Route traffic, enforce access rules |
| **DMZ (Demilitarized Zone)** | Buffer network for public-facing servers |
| **Remote Access Gateways/VPNs** | Secure entry points for remote employees |

## Network Perimeter in Small Enterprise

```
Internet
    ↓
Firewall (between Internet and internal LAN)
    ↓
DMZ (Web server for public access)
    ↓
Internal Network (AD, file, database servers)
    ↓
VPN Gateway (remote employees)
```

## Role of Security Analyst

**Monitoring the Perimeter Means:**
- Reviewing firewall logs for blocked/allowed connections
- Identifying scanning attempts
- Detecting brute-force login attempts
- Flagging unusual outbound traffic
- Understanding what should/should not be exposed

# 5. Monitoring the Perimeter

## How to Monitor the Perimeter

**Tools Used:**
- Firewalls
- Intrusion Detection/Prevention Systems (IDS/IPS)
- Access Control

**Benefits:**
- Spot early-stage attacks (port scans, brute-force)
- Detect misconfigurations
- Identify outbound traffic anomalies

## Scenario 1: Port Scanning

**Firewall Log:**
```
2025-09-22 08:30:04 ALLOW TCP 198.51.100.45:49876 -> 10.0.0.51:80
2025-09-22 08:30:05 BLOCK TCP 203.0.113.10:50001 -> 10.0.0.20:21
2025-09-22 08:30:06 BLOCK TCP 203.0.113.10:50002 -> 10.0.0.20:22
2025-09-22 08:30:07 ALLOW TCP 192.0.2.115:51235 -> 10.0.0.50:443
2025-09-22 08:30:08 BLOCK TCP 203.0.113.10:50003 -> 10.0.0.20:23
2025-09-22 08:30:09 BLOCK TCP 203.0.113.10:50004 -> 10.0.0.20:25
2025-09-22 08:30:10 ALLOW TCP 198.51.100.92:51111 -> 10.0.0.50:443
2025-09-22 08:30:11 BLOCK TCP 203.0.113.10:50005 -> 10.0.0.20:53
```

**Pattern Detected:**
- Same external IP (203.0.113.10)
- Multiple ports on same internal machine
- Rapid succession

**Analyst's Verdict:** Classic port scan. Attacker looking for open service to target.

## Scenario 2: Web Application Attack

**WAF Log:**
```
timestamp=2025-09-22T09:14:44Z src_ip=192.0.2.130 action=ALLOW request="GET /index.html"
timestamp=2025-09-22T09:14:45Z src_ip=198.51.100.92 action=ALLOW request="GET /products.php?id=9"
timestamp=2025-09-22T09:14:46Z src_ip=[REDACTED] action=BLOCK request="GET /search.php?q=<script>alert('XSS')</script>" rule_id=941100 attack_type="XSS"
timestamp=2025-09-22T09:15:42Z src_ip=[REDACTED] action=BLOCK request="GET /../../../../etc/passwd" rule_id=930120 attack_type="Directory Traversal"
```

**Analysis:**
- Mix of ALLOW and BLOCK actions
- Filter `action=BLOCK` to find threats
- WAF provides attack type information

**Attack Types Detected:**
- SQL Injection
- XSS (Cross-Site Scripting)
- Directory Traversal

**Analyst's Verdict:** WAF successfully identifies and blocks multiple web attacks. High-confidence alert that attacker is actively targeting website.

## Scenario 3: VPN Brute-Force

**VPN Gateway Log:**
```
2025-09-22 10:12:11 FAILED_AUTH TCP [REDACTED]:31245 -> 10.0.0.1:443 (user 'admin')
2025-09-22 10:12:15 FAILED_AUTH TCP [REDACTED]:31248 -> 10.0.0.1:443 (user 'admin')
2025-09-22 10:12:21 SUCCESS_AUTH TCP 198.51.100.88:41233 -> 10.0.0.1:443 (user 'b.jones')
2025-09-22 10:12:08 FAILED_AUTH TCP [REDACTED]:31249 -> 10.0.0.1:443 (user 'guest')
2025-09-22 10:12:09 FAILED_AUTH TCP [REDACTED]:31250 -> 10.0.0.1:443 (user 'user')
```

**Analysis:**
- Flooded with FAILED_AUTH events
- A few successful logins are normal
- Filter by source IP reveals attack

**Pattern Detected:**
- One suspicious IP
- Hundreds of failed attempts
- Attempting common usernames (admin, root, test)

**Analyst's Verdict:** Brute-force attack against VPN gateway. Attacker using list of common usernames to find valid account.

## Key Takeaways

**Look for Patterns:**

| Pattern | Indicates |
|---------|-----------|
| Repetition from one source to many destinations | Scanning |
| Repetition from one source to one destination | Brute-forcing |
| Traffic at perfect, regular intervals | Malware beaconing |

**Context is Key:**
- IDS alert with attack type > simple firewall block
- Understand what's normal vs suspicious

# 8. Quick Revision

## Network Components

| Component | Purpose | Security Risk |
|-----------|---------|---------------|
| Workstations | Employee work | Phishing, malware entry |
| File/Database Servers | Data storage | Ransomware, exfiltration |
| Application Servers | Web/email/VPN | Exploitation, scanning |
| AD/Authentication | Identity management | Privilege escalation |
| Routers/Switches | Traffic routing | Man-in-the-Middle |
| Firewalls | Traffic filtering | Misconfiguration |

## Log Types

| Log Type | Source | What It Shows |
|----------|--------|---------------|
| Host-Centric | Individual devices | Process execution, user activity |
| Network-Centric | Network appliances | Traffic flow, connections |

## Perimeter Security

| Component | Purpose |
|-----------|---------|
| Firewall | Filter traffic |
| Router | Route traffic |
| DMZ | Public-facing servers |
| VPN | Remote access |

## Attack Patterns

| Pattern | Indicates |
|---------|-----------|
| One source → many destinations (blocked) | Port scanning |
| One source → one destination (many attempts) | Brute-force |
| Regular interval traffic | C2 beaconing |
| Large outbound transfers | Data exfiltration |

## Key Commands

```bash
# Find blocked connections
cat firewall.log | grep "BLOCK" | head

# Count top attacking IPs
cat firewall.log | grep "BLOCK" | cut -d' ' -f5 | cut -d: -f1 | sort -nr | uniq -c

# Check allowed connections from IP
cat firewall.log | grep [IP] | grep "ALLOW"

# Count failed VPN attempts
cat vpn_auth.log | grep FAIL | cut -d' ' -f3 | sort -nr | uniq -c

# Find C2 beaconing
cat ids_alerts.log | grep C2

# Find data exfiltration
cat ids_alerts.log | grep "Data Exfiltration"
```

## Common Ports to Monitor

| Port | Service | Risk |
|------|---------|------|
| 21 | FTP | Data exfiltration |
| 22 | SSH | Brute-force, lateral movement |
| 23 | Telnet | Insecure |
| 25 | SMTP | Email abuse |
| 53 | DNS | Tunneling |
| 80 | HTTP | Web attacks |
| 443 | HTTPS | Web attacks, C2 |
| 445 | SMB | Lateral movement, ransomware |
| 3389 | RDP | Brute-force, lateral movement |
| 4444 | C2 | C2 beaconing |

## Important Terms

| Term | Definition |
|------|------------|
| C2 (Command & Control) | Attacker controlling compromised host |
| Lateral Movement | Attacker moving between systems |
| Data Exfiltration | Stealing data from network |
| DMZ (Demilitarized Zone) | Buffer network segment |
| IDS (Intrusion Detection System) | Monitors for attacks |
| IPS (Intrusion Prevention System) | Blocks attacks |
| WAF (Web Application Firewall) | Protects web applications |

## Investigation Mindset

1. **Start with blocked requests** - What's being stopped?
2. **Identify suspicious IPs** - Who's causing most blocks?
3. **Check for allowed requests** - Did attacker get through?
4. **Look for credential access** - Brute-force successes?
5. **Find lateral movement** - Internal scanning?
6. **Detect C2** - Beaconing patterns?
7. **Confirm exfiltration** - Large data transfers?

---

## Analysing Tips

1. **Understand the attack chain** - Recon → Initial Access → Lateral Movement → C2 → Exfiltration
2. **Learn log correlation** - Combine firewall, IDS, and VPN logs
3. **Practice command-line analysis** - Filter, sort, count, uniq
4. **Know your network assets** - IP addresses, roles, criticality
5. **Recognize patterns** - Scanning, brute-forcing, beaconing
6. **Use Splunk for large datasets** - Faster, more visual
7. **Context is key** - Understand normal behavior to spot anomalies
