# Intro to Logs

## 1. What Are Logs?

### Definition
- Logs are records of events occurring on systems, applications, networks, or devices.
- They act as a historical timeline of activities.

### Why Logs Matter
- Detect malicious activity
- Investigate incidents
- Troubleshoot issues
- Monitor performance
- Meet compliance requirements

### Analogy
- Tree rings show a tree's history.
- Logs show a system's history.

---

## 2. Components of a Log Entry

A log entry usually contains:

| Component | Description |
|------------|------------|
| Timestamp | When the event occurred |
| Source | System or application generating the log |
| Event Type | What happened |
| Details | User, IP address, action, status, etc. |

### Example

```text
May 31 12:34:56 WEBSRV-02 CRON[2342593]: Job executed
```

- Timestamp → May 31 12:34:56
- Host → WEBSRV-02
- Process → CRON
- Event → Job execution

---

## 3. Logs as Evidence

A single log entry often provides little value.

Multiple logs combined provide context and help answer:

### What happened?
- Login attempt
- File access
- HTTP request

### When did it happen?
- Exact timestamp

### Where did it happen?
- Hostname or IP address

### Who performed it?
- User account or device

### Was it successful?
- Success or failure status

### What was the result?
- Access granted
- File modified
- Malware executed

## 5. Log Formats

A log format defines how log data is stored and structured.

### Categories

1. Semi-Structured
2. Structured
3. Unstructured

---

## 6. Semi-Structured Logs

### Characteristics
- Predictable structure
- Some free-form text

### Syslog Example

```text
May 31 12:34:56 WEBSRV-02 CRON[1234]: Job executed
```

### Windows EVTX Example

```text
TimeCreated    EventID
31/05/2023     16384
```

---

## 7. Structured Logs

### Characteristics
- Fixed fields
- Easy to parse
- Machine-friendly

### CSV

```csv
time,user,status
2023-05-31,admin,200
```

### JSON

```json
{
  "user":"admin",
  "status":200
}
```

### XML

```xml
<user>admin</user>
```

### W3C ELF

```text
date time c-ip cs-method sc-status
```

Commonly used by:
- IIS Web Server

---

## 8. Unstructured Logs

### Characteristics
- Human-readable
- Harder to parse automatically

### NCSA Common Log Format (CLF)

```text
34.253.159.159 - admin [date] "GET /"
```

Used by:
- Apache

### NCSA Combined Log Format

```text
IP User Request Status Referrer User-Agent
```

Used by:
- Nginx

## 12. Time Synchronization (NTP)

### Why Important?

Without synchronized clocks:

```text
Firewall: 10:01
Server:   09:58
IDS:      10:04
```

Investigations become difficult.

### Solution

**Network Time Protocol (NTP)**

```bash
ntpdate pool.ntp.org
```

### Benefits
- Accurate timestamps
- Reliable investigations
- Easier event correlation

## 15. rsyslog

### Purpose
Linux logging service.

### Functions
- Collect logs
- Filter logs
- Store logs
- Forward logs

### Example Rule

```text
:programname, isequal, "sshd" /var/log/websrv-02/rsyslog_sshd.log
```

### Meaning
- Capture all SSH daemon logs
- Save them to a dedicated file

## 17. Log Retention

### Hot Storage
**3–6 Months**

- Fastest access
- Near real-time search

### Warm Storage
**6 Months – 2 Years**

- Slower access
- Still searchable

### Cold Storage
**2–5 Years**

- Archived
- Compressed
- Used for historical investigations

### Memory Trick

```text
Hot   = Recent
Warm  = Older
Cold  = Archive
```

---
