# Putting It All Together – Quick Notes

## How a Website Loads
When a website is requested:

1. **DNS** resolves the domain name into an IP address.
2. The browser communicates with the web server using **HTTP/HTTPS**.
3. The server responds with resources such as:
   - HTML
   - CSS
   - JavaScript
   - Images
4. The browser renders the webpage for the user.

---

## Load Balancer

A **Load Balancer** sits in front of multiple servers and distributes incoming requests.

### Functions
- Distributes traffic across multiple servers
- Prevents server overload
- Provides failover if a server becomes unavailable

### Common Algorithms
- **Round Robin** → Requests are distributed sequentially
- **Weighted** → Sends requests to the least busy server

### Health Check
Load balancers continuously monitor servers using **health checks** and stop sending traffic to failed servers.

---

## CDN (Content Delivery Network)

A **CDN** stores copies of static content on servers located around the world.

### Static Content Examples
- Images
- CSS
- JavaScript
- Videos

### Benefits
- Faster loading times
- Reduced server load
- Improved user experience

The CDN serves files from the nearest geographical server.

---

## Databases

Databases store and manage website data.

### Common Uses
- User accounts
- Blog posts
- Application data
- Website settings

### Popular Databases
- MySQL
- MSSQL
- PostgreSQL
- MongoDB

---

## WAF (Web Application Firewall)

A **WAF** sits between users and the web server.

### Purpose
- Blocks malicious requests
- Protects against web attacks
- Helps prevent DoS/DDoS attempts

### Features
- Request inspection
- Bot detection
- Rate limiting
- Attack filtering

---

# How Web Servers Work

## Web Server

A **Web Server** is software that listens for requests and serves web content using HTTP/HTTPS.

### Common Web Servers
- Apache
- Nginx
- IIS
- Node.js

### Default Web Roots
| Server | Default Directory |
|----------|------------------|
| Apache / Nginx | `/var/www/html` |
| IIS | `C:\inetpub\wwwroot` |

---

## Virtual Hosts

**Virtual Hosts** allow a single web server to host multiple websites.

### How It Works
The server checks the **Host** header in the HTTP request and serves the matching website.

### Benefits
- Multiple domains on one server
- Separate website directories
- Efficient resource usage

---

## Static vs Dynamic Content

### Static Content
Content that remains unchanged.

**Examples**
- Images
- CSS
- JavaScript
- Fixed HTML pages

### Dynamic Content
Content generated based on user requests or data.

**Examples**
- Search results
- User dashboards
- Blog feeds
- E-commerce pages

Dynamic content is generated by backend code before being sent to the browser.

---

## Frontend vs Backend

### Frontend (Client-Side)
Everything visible to the user in the browser.

**Technologies**
- HTML
- CSS
- JavaScript

### Backend (Server-Side)
Processes requests and generates responses behind the scenes.

**Responsibilities**
- Business logic
- Database interaction
- Authentication
- Data processing

Users cannot directly see backend code.

---

## Backend Languages

Common server-side languages include:

- PHP
- Python
- Ruby
- Node.js
- Perl

### Capabilities
- Process user input
- Communicate with databases
- Generate dynamic content
- Connect to external services

---

## Key Takeaways

- DNS converts domain names into IP addresses.
- HTTP/HTTPS enables browser-server communication.
- Load Balancers distribute traffic and perform health checks.
- CDNs speed up websites by serving static files from nearby locations.
- Databases store application data.
- WAFs protect websites from attacks.
- Web servers deliver website content to users.
- Virtual Hosts allow multiple websites on one server.
- Static content remains unchanged, while dynamic content is generated on demand.
- Frontend is visible to users; backend operates behind the scenes.
