# Cloud Computing Fundamentals – Quick Notes

## Overview

Cloud Computing provides computing resources over the internet instead of relying on a single physical computer.

It is built on technologies such as:

- Virtualization
- Containers

Cloud computing allows applications to be accessible, scalable, reliable, and cost-effective.

---

## Why Cloud Computing?

Running applications on a personal computer creates several challenges:

- Limited hardware resources
- Poor global accessibility
- Downtime when the computer is offline
- Difficulty handling many users simultaneously
- Expensive upgrades

Cloud computing solves these issues by providing on-demand resources from large data centers.

---

## Evolution of Servers

### Physical Servers
- One server for one application
- Expensive
- Poor resource utilization

### Virtualization
- Multiple virtual machines on one server
- Better hardware utilization

### Containers
- Lightweight application environments
- Faster and more efficient deployment

### Cloud Computing
- Resources delivered over the internet
- Scalable and globally available

---

# Cloud Characteristics

## Scalability

The ability to increase or decrease resources based on demand.

**Example**
- Handling sudden traffic spikes
- Supporting more users during peak hours

---

## On-Demand Self-Service

Resources can be created or removed instantly without purchasing hardware.

---

## Pay-As-You-Go

You only pay for the resources you use.

### Benefits
- No large upfront investment
- Cost optimization
- Flexible budgeting

---

## Security

Cloud providers secure the underlying infrastructure through:

- Physical security
- Network security
- Monitoring systems
- Access controls

---

## High Availability

Applications remain operational even if some infrastructure components fail.

---

## Global Access

Users can access applications from anywhere in the world.

---

# Cloud Deployment Models

## Public Cloud

Infrastructure is owned and managed by a cloud provider.

### Characteristics
- Most common deployment model
- Easy to scale
- Cost-effective
- No infrastructure maintenance

### Examples
- AWS
- Azure
- Google Cloud

---

## Private Cloud

Infrastructure is dedicated to a single organization.

### Common Users
- Banks
- Government agencies
- Healthcare organizations

### Benefits
- Greater control
- Enhanced security
- Compliance support

---

## Hybrid Cloud

Combines Public and Private Cloud environments.

### Benefits
- Flexibility
- Improved scalability
- Sensitive data remains private

### Common Use Cases
- E-commerce platforms
- Large enterprises

---

# Cloud Service Models

## IaaS (Infrastructure as a Service)

Provides virtual infrastructure resources.

### Includes
- Virtual servers
- Storage
- Networking

### Customer Manages
- Operating system
- Applications
- Security configurations

### Examples
- AWS EC2
- Azure Virtual Machines

---

## PaaS (Platform as a Service)

Provides a platform for application development and deployment.

### Provider Manages
- Hardware
- Operating system
- Runtime environment

### Developer Focuses On
- Writing and deploying applications

### Benefits
- Faster development
- Less infrastructure management

---

## SaaS (Software as a Service)

Complete software delivered over the internet.

### Characteristics
- Ready to use
- No installation required
- Provider manages everything

### Examples
- Gmail
- Zoom
- Microsoft 365

# AWS Basics

## EC2 (Elastic Compute Cloud)

An EC2 instance is a virtual computer running in the cloud.

### Features
- CPU
- RAM
- Storage
- Networking

### Uses
- Web servers
- Applications
- Cyber security labs
- Databases

---

## Instance Types

Instance types define the hardware specifications of an EC2 instance.

### General Rule

| Instance Size | Power | Cost |
|--------------|--------|------|
| Small | Lower | Lower |
| Large | Higher | Higher |

### Examples
- t3.micro
- t3a.small
- m5.large

---

## Region

A Region is a geographical location where cloud resources are deployed.

### Benefits
- Lower latency
- Regulatory compliance
- Disaster recovery planning

Examples:
- North America
- Europe
- Asia Pacific

---

# Cloud Cost Optimization

One major cloud advantage is cost control.

### Reduce Costs By

- Stopping unused instances
- Scaling down resources
- Removing unnecessary services

### Benefit

Pay only for resources actively being used.

---

# Key Takeaways

- Cloud computing delivers resources over the internet.
- Virtualization and containers are the foundation of cloud platforms.
- Scalability allows applications to handle changing demand.
- Public Cloud is the most commonly used deployment model.
- IaaS provides infrastructure, PaaS provides platforms, and SaaS provides complete software.
- AWS is the leading cloud provider.
- EC2 instances are virtual computers in the cloud.
- Regions determine where resources are deployed.
- Cloud platforms enable global availability, flexibility, and cost optimization.
